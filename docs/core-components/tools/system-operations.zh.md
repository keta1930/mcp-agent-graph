# 系统操作工具

让 Agent 查询系统资源和配置。

## 什么是系统操作工具

系统操作工具让 Agent 了解系统中可用的资源。Agent 可以查询模型、MCP 服务、工作流和提示词，避免重复创建，选择合适的工具。

## 核心能力

| 能力 | 说明 |
|------|------|
| **模型查询** | 查看所有可用的模型名称 |
| **MCP 管理** | 查看 MCP 服务器状态和工具列表 |
| **工作流查询** | 浏览和查看工作流配置 |
| **提示词查询** | 查找和读取提示词内容 |
| **工具清单** | 查看所有系统工具的分类和功能 |

## 工作原理

Agent 在需要了解系统资源时调用这些工具。例如，创建新 Agent 前查询可用模型，或设计工作流前查看已有的工作流。

**示例流程：**

```
用户："帮我创建一个代码审查 Agent"
↓
Agent 调用 list_all_models
查看可用模型列表
↓
Agent 调用 list_all_prompts
检查是否有相关提示词
↓
Agent 使用查询结果设计 Agent
```

## 可用工具

### 模型查询

| 工具 | 功能 |
|------|------|
| `list_all_models` | 列出所有已配置的模型名称 |

**使用场景：** Agent 创建、模型选择、配置验证

### MCP 服务查询

| 工具 | 功能 |
|------|------|
| `list_all_mcps` | 列出所有 MCP 服务器、连接状态和工具概览（已连接服务器显示前 3 个工具） |
| `get_mcp_details` | 获取指定 MCP 服务器的完整配置和所有工具列表 |

**使用场景：** 工具选择、MCP 状态检查、能力评估

### 工作流查询

| 工具 | 功能 |
|------|------|
| `list_all_graphs` | 列出所有工作流及其描述 |
| `get_graph_details` | 获取指定工作流的完整配置（包括所有节点和连接关系） |

**使用场景：** 工作流设计、参考已有流程、避免重复创建

### 提示词查询

| 工具 | 功能 |
|------|------|
| `list_all_prompts` | 列出所有提示词及其分类 |
| `get_prompt_content` | 获取指定提示词的完整内容 |

**使用场景：** 提示词复用、内容参考、模板选择

### 工具清单

| 工具 | 功能 |
|------|------|
| `list_system_tools` | 列出所有系统工具的分类、名称和功能说明 |

**使用场景：** 了解系统能力、工具选择、Agent 配置

## 使用建议

**查询前置：** 在创建新资源前先查询已有资源

**按需查询：** 只查询当前任务需要的信息

**缓存结果：** 在同一对话中复用查询结果

**组合使用：** 先用列表工具浏览，再用详情工具深入了解

## 权限与访问

- 查询结果仅包含您有权访问的资源
- MCP 连接状态反映当前实时状态
