# Task（任务）

定时自动执行 Graph。

## 什么是 Task

Task 让您按时间计划自动运行 Graph。支持单次执行或 cron 周期执行。

## 任务管理

用户可通过界面管理任务：

| 操作 | 作用 | 效果 |
|------|------|------|
| **查看列表** | 查看所有任务及状态、计划、执行统计 | 监控自动化全局 |
| **查看详情** | 查看完整执行历史和时间戳 | 审计工作流运行记录 |
| **暂停任务** | 临时停止执行 | 任务恢复前不会运行 |
| **恢复任务** | 重启暂停的任务 | 按计划继续执行 |
| **删除任务** | 永久移除任务 | 不再执行 |

**任务状态：**

| 状态 | 含义 |
|------|------|
| Active（活跃） | 任务已调度，将会执行 |
| Paused（暂停） | 任务存在但不会执行，直到恢复 |
| Completed（已完成） | 单次任务已执行 |
| Error（错误） | 任务遇到问题（如时间已过期） |


## 执行说明

**并发实例：** 设置 `execution_count` 可在每次触发时运行多个 Graph 实例，每个有独立的 `conversation_id`。

**后台模式：** 任务始终后台执行，在会话历史中查看结果。

**历史跟踪：** 每次触发记录时间戳和所有 `conversation_id`

## 如何使用

**创建任务：** 进入任务管理页面，点击"创建任务"，选择 Graph、设置调度和输入。

**查看任务：** 任务列表显示所有任务的状态、下次运行时间和执行历史。

**管理状态：** 使用状态下拉菜单暂停、恢复或完成任务。

**查看结果：** 点击执行历史可查看每次触发的会话结果。

## 注意事项

- 所有时间使用服务器本地时区
- 失败任务转为 `error` 状态并停止
- 输入文本在创建时固定，无法动态修改
- 活跃任务在重启时自动重新加载
- 要修改任务，需删除后重新创建
